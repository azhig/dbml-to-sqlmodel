# Руководство по интерактивному CLI

## Запуск

Просто запустите команду без аргументов:

```bash
uv run dbml-crud
```

Или через Makefile:

```bash
make cli
```

## Как пользоваться

### Главное меню

После запуска вы увидите приветственный баннер и главное меню:

```
╭──────────────────────────────────────────────╮
│ Welcome                                      │
│ 🚀 DBML to CRUD Generator                    │
│                                              │
│ Modern FastAPI + SQLModel + FastCRUD         │
│ generator from DBML schemas                  │
╰──────────────────────────────────────────────╯

? Что вы хотите сделать?
  ⚙️  Настройки
❯ 🎯 DBML → Code
  🔁 Code → DBML
  📊 Report
  ❌ Выход
```

### Навигация

- **↑↓** (стрелки вверх/вниз) - перемещение по меню
- **Enter** - выбрать пункт
- **Ctrl+C** - отменить текущую операцию и вернуться в меню
- **Y/N** - ответить на вопросы да/нет

## Команды

### ⚙️ Настройки

Управляет параметрами работы (путь к DBML, директория вывода, поведение preview):

1. Выберите пункт "Настройки"
2. Измените значения или сбросьте по умолчанию

### 📊 Report

Показывает список генерируемых файлов и различия с текущим `output/`:

1. Выберите пункт "Report"
2. Используются настройки из `.dbml_to_crud`

**Что показывает:**
- Таблица статусов для `models/*/model.py`
- Diff для измененных model.py
- Полный контент новых model.py

### 🎯 DBML → Code

**Основная особенность:** автоматически показывает diff перед генерацией!

Workflow:

1. Выберите пункт "DBML → Code"
2. **Если директория существует:**
   - Автоматически показывается предпросмотр (diff)
   - Запрашивается подтверждение: "Применить изменения?"
   - Можно отменить, нажав `N`
3. Генерация и запись файлов

**Защита пользовательских файлов:**
- Файлы с маркером `# USER_MODIFIED` автоматически защищаются
- В preview они отображаются как 🔒 Protected
- Они пропускаются при генерации (если не указан --force)

### 🔁 Code → DBML

Генерирует DBML из `output/models/`, показывает diff и предлагает сохранить:

1. Выберите пункт "Code → DBML"
2. Используются настройки из `.dbml_to_crud`
3. Показывается diff DBML перед сохранением

### ❌ Выход

Завершает работу CLI.

## Continuous Workflow

После выполнения команды CLI возвращается в меню.

Это позволяет выполнять несколько операций подряд:

1. Report - посмотреть файлы и несоответствия
2. DBML → Code - применить изменения
3. Report - убедиться что всё правильно
4. Exit - выйти

## Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| ↑↓ | Навигация по меню |
| Enter | Выбрать/подтвердить |
| Y | Да (yes) |
| N | Нет (no) |
| Ctrl+C | Отменить/прервать |
| Ctrl+D | Выход (в некоторых контекстах) |

## Советы

### 1. Используйте автоматический preview

```bash
# Хорошая практика:
1. Report - смотрим что изменится
2. DBML → Code - применяем изменения
```

### 2. Защищайте кастомные файлы

Добавьте в начало файла:

```python
# USER_MODIFIED
# My custom changes

# ... your code ...
```

### 3. Continuous Workflow для сложных задач

```bash
1. Report - посмотреть файлы
2. DBML → Code - применить
3. Report - убедиться что всё правильно
```

## Troubleshooting

### CLI не запускается

```bash
# Переустановить зависимости
uv sync
```

### Интерактивное меню не работает

Убедитесь что терминал поддерживает интерактивный ввод:

```bash
# Проверить переменную окружения
echo $TERM

# Должно быть что-то вроде: xterm-256color
```

### Arrows keys не работают

Некоторые терминалы требуют особой настройки. Попробуйте:

```bash
export TERM=xterm-256color
uv run dbml-crud
```

## Альтернатива: Прямые команды

Если интерактивный режим не работает, можно использовать прямые команды:

```bash
# Report
uv run dbml-crud info schemas/schema.dbml -o output

# Preview (only model.py files)
uv run dbml-crud preview schemas/schema.dbml -o output

# Generate
uv run dbml-crud generate schemas/schema.dbml -o output
```
